DNS

To provide a link between domain names and IP addresses, a system called the Domain Name System (DNS) handles all requests through allocated servers. For example, a DNS server would be able to link www.Google.com to a proper IP address, allowing a user to connect to the website. There is not only one DNS server, and because of this, not all DNS servers have all of the internet’s many IP addresses. To solve this problem, multiple DNS servers will “talk” to each other, and find the IP address related to a given domain name in lightning fast times.

DNS was originally created to be an open and public communication protocol for government and educational institutions. Because of this, DNS servers are highly susceptible to cyber attacks.

One of these attacks is called DNS Spoofing. DNS Spoofing is a method of interfering with a DNS server where a hacker attempts to change the IP address of a specific or multiple domain names. This allows a hacker to cause the server to send people to a compromised website, most likely for a malicious intent, whether that be to install a virus or to scam visitors.
