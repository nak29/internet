HTTP and HTTPS protocols

HTTP stands for Hypertext Transfer Protocol, and it’s how web browsers and servers communicate to each other. Most of the “dialogue” going on between the two are made up of “GET requests”, which simply put is the web browser requesting data from the server. HTTP is mostly made up of the web browser requesting data from the server and not the other way around. However, the browser can sometimes send data, such as login information details.

HTTPS is an extension of HTTP called Hypertext Transfer Protocol Secure. The main difference between the two is that HTTPS confirms the webpage you are on uses public key encryption to exchange data. HTTPS primarily uses two security protocols named SSL (secure sockets layer) and TLS (transport layer security). These two are intricate enough to have their own dedicated webpages explaining them in length, but for now, just know they use encryption to keep us safe.
